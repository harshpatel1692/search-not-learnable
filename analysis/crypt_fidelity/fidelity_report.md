## Checker semantics / caveats (for reading the table honestly)

- Domain references (`X in {..}`, enumd pre-domains, eliml free sets) are checked with SUBSET semantics against a conservative tracked state (10 digits minus visible pins/exclusions, minus 0 for leading letters): a stated domain narrower than tracked is accepted (engine-style narrowing is not always visible), wider = transcription fail. So the T numbers for kill/pin witnesses are an UPPER bound on copy fidelity.
- Range-kill bounds are checked for SOUNDNESS (a true bound), matching the renderer's sound-interval bounds; a true-but-non-killing bound (e.g. `|a-b| <= 89` against a 2-digit target) fails only the VERDICT axis. kill/range A=97.1 vs V=51.0 means: bounds are real arithmetic, but half the range-kill conclusions do not follow from them.
- kill/sign V=100 but T=95.1: the sign-vs-glyph logic itself is always applied correctly, but in ~5% of lines the model picks/claims an example equation whose RHS does NOT have the stated sign property (fabricated premise -> transcription fail, verdict vacuously consistent).
- bail/stall-interchangeable T=0/70 is dominated by 2 degenerate-loop transcripts that re-guess an already-pinned symbol dozens of times (s36..s105 identical lines); it is a loop pathology, not 70 independent errors.
- Exactly 1 of the 100 prompts (1d15b7e6) has a true op-glyph/digit-glyph overlap (`@`), which the r8 grammar never taught; its union/disjoint fails are charged to the model in this table.
- try-counter rows are the sN counters audited as their own one-check records (monotone +1); 2365/2365 correct.
- `verify()`-level note: greedy boxed == gold on only 4/100, consistent with eval_oracle's ~.05.
